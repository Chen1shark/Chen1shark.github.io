import java.util.Scanner;

public class ListNode {
    int val;
    ListNode next;
    
    ListNode(int x) {
        val = x;
        next = null;
    }
    
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        // 示例：创建链表 1->2
        ListNode head = new ListNode(1);
        head.next = new ListNode(2);
        
        // 调用静态方法打印链表
        printList(head);
        
        // 调用非静态方法获取链表长度
        int length = head.getLength();
        System.out.println("链表长度: " + length);
    }
    
    // 静态方法，若非静态方法，则需要先创建对象
    public static void printList(ListNode head) {
        ListNode current = head;
        while (current != null) {
            System.out.print(current.val);
            if (current.next != null) {
                System.out.print(" -> ");
            }
            current = current.next;
        }
        System.out.println();
    }
    
    // 非静态方法示例：获取链表长度
    public int getLength() {
        int length = 0;
        ListNode current = this;
        while (current != null) {
            length++;
            current = current.next;
        }
        return length;
    }
}

// 链表两两交换 - 正确做法
public ListNode swapPairs(ListNode head) {
    ListNode dummy = new ListNode(0);    // 虚拟头节点
    dummy.next = head;
    ListNode prev = dummy;               // prev 指向 dummy
    ListNode curr = head;                // curr 指向实际头节点
    
    while (curr != null && curr.next != null) {
        ListNode next = curr.next;       // 保存下一个节点
        curr.next = next.next;           // 当前节点指向下下个
        next.next = curr;                // 下个节点指向当前
        prev.next = next;                // 前驱节点指向交换后的头
        
        prev = curr;                     // 移动指针
        curr = curr.next;
    }
    return dummy.next;
}

关键点：所有指针（dummy, prev, curr）操作同一个链表对象，修改会相互影响。

// 错误示例 - 不要这样做！
public ListNode swapPairsWrong(ListNode head) {
    ListNode dummy1 = new ListNode(0);   // 独立虚拟头节点1
    ListNode dummy2 = new ListNode(0);   // 独立虚拟头节点2
    dummy1.next = head;
    dummy2.next = head;                  // ❌ dummy2.next 固定指向原头节点
    
    ListNode curr = dummy1.next;         // curr 指向原链表 head
    while (curr != null && curr.next != null) {
        ListNode next = curr.next;
        curr.next = next.next;           // 交换节点
        next.next = curr;                // 交换节点
        // ❌ 致命错误：没有更新 dummy2.next！
        // 正确做法应该有：dummy2.next = next;
        curr = curr.next.next;
    }
    return dummy2.next;                  // ❌ 返回错误位置，链表截断
}

算法题中的核心区别：

  • 正确做法：一个虚拟头节点 + 多个工作指针，操作同一个链表
  • 错误做法：多个独立虚拟头节点，结果完全错误

⚠️ 方法二的致命问题：

// 原链表：1→2→3→4
ListNode dummy1 = new ListNode(0);
ListNode dummy2 = new ListNode(0);
dummy1.next = head;  // dummy1.next -> node1
dummy2.next = head;  // dummy2.next -> node1 (固定指向node1)

// 两两交换后链表变成：2→1→4→3
// 但 dummy2.next 仍然指向 node1（现在在第二个位置）
return dummy2.next;  // 返回：1→4→3 ❌ 丢失了头节点！

为什么方法二错误:

  • dummy2.next 固定指向原头节点，不知道交换后的新头节点
  • 两两交换会改变头节点位置，但 dummy2 无法感知这个变化
  • 结果：返回错误的起始位置，导致链表截断

高频应用场景：

  • 链表反转、两两交换、删除节点等
  • 需要记录前驱节点时，用 prev = dummy 而不是创建新的虚拟节点

// 1. Lambda表达式 - 最简洁
list.sort((a, b) -> b - a);                    // 直接写比较逻辑
PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> b - a);

// 2. Comparator - 传统接口
list.sort(new Comparator<Integer>() {
    @Override
    public int compare(Integer a, Integer b) {
        return b - a;
    }
});

// 3. Comparator.comparing - 方法引用（最强大）
list.sort(Comparator.comparing(Person::getAge));          // 按年龄升序
list.sort(Comparator.comparing(Person::getAge).reversed()); // 按年龄降序
list.sort(Comparator.comparingInt(Person::getAge));        // 专门int类型（避免拆箱）

// 4. 多字段排序
list.sort(Comparator
    .comparing(Person::getAge)
    .thenComparing(Person::getName)
);
三者区别说明
  • Lambda：(a, b) -> b - a - 最简洁，直接写比较逻辑
  • Comparator：new Comparator<>() - 传统接口，需要重写compare方法
  • comparing：Comparator.comparing() - 方法引用，最强大，支持链式调用

使用建议:

  • 简单比较用Lambda
  • 复杂逻辑用Comparator
  • 对象属性排序用comparing
compareTo / Comparator / comparing / Comparable 区别与使用场景

  • Comparable & compareTo（类内部默认排序）

    • 类实现 Comparable<T>，重写 compareTo，定义"唯一默认排序"
    • 调用方式：a.compareTo(b)；常见于 String、Integer
    • 适用：类有天然唯一顺序（如时间戳、ID、年龄），不需要多种排序
    • 只有一个抽象方法：compareTo(T o)（函数式接口）
    1
2
3

    class Person implements Comparable<Person> {
    int age;
    public int compareTo(Person o) { return this.age - o.age; }
}

Collections.sort(personList); // 按年龄升序

  • Comparator（类外部可选排序）

    • 函数式接口，compare(a, b)，可有多个实现，不改动类本身
    • 形式：匿名内部类或 Lambda：(a, b) -> ...
    • 适用：需要多种排序策略（年龄、姓名、复合字段等）
    1

    personList.sort((a, b) -> b.age - a.age); // 年龄降序（Lambda）

  • Comparator.comparing / thenComparing（链式工具，推荐）

    • 语法糖，本质是 Comparator，支持链式组合与类型安全方法引用
    • 适用：对象属性排序、多字段排序，写法最简洁可读
    1
2

    personList.sort(Comparator.comparing(Person::getAge)
                           .thenComparing(Person::getName));

  • 选用指南

    1. 只有一个天然顺序 → 在类上实现 Comparable/compareTo
    2. 需要多种排序策略 → 用 Comparator（Lambda 或匿名类）
    3. 需要属性和多字段排序 → 用 Comparator.comparing().thenComparing()
