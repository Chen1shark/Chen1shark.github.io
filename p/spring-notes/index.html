Spring Bean 核心定义

Spring Bean 是指由 Spring IoC 容器负责生命周期管理的对象。这些对象通过配置（如注解 @Component、@Service，或 XML/JavaConfig）被注册到容器中，由容器完成实例化、依赖注入（DI）、初始化、使用和销毁的全过程。

所有业务组件，比如 Controller、Service、Repository 等，本质上都是 Spring Bean。它们之间的依赖关系不是硬编码 new 出来的，而是由容器自动装配，从而实现控制反转（IoC）和松耦合。

简单说：Bean = 对象 + 生命周期 + 依赖关系，全部交给 Spring 容器管理。

Spring Bean 的核心特性

  • 容器托管：生命周期完全由 Spring IoC 容器控制，无需开发者手动创建和销毁
  • 依赖注入：Bean间依赖由容器自动装配，降低组件耦合度
  • 配置灵活：支持 注解、XML、JavaConfig等多种配置方式定义 Bean
  • 可扩展：支持生命周期回调、后置处理器等自定义扩展逻辑

Bean 的生命周期

1. 实例化

容器通过反射创建 Bean 实例（调用构造方法）；

2. 属性填充

为 Bean 注入依赖（如 @Autowired、setter 注入、构造器注入）；

3. 初始化（核心阶段）

  • 执行 Aware 接口方法（如 BeanNameAware、BeanFactoryAware）；

  • 执行 BeanPostProcessor 前置处理（postProcessBeforeInitialization）；

  • 执行初始化方法（@PostConstruct → InitializingBean.afterPropertiesSet() → 自定义 init-method）；

  • 执行 BeanPostProcessor 后置处理（postProcessAfterInitialization）；

4. 使用
5. 销毁

容器关闭时，执行销毁方法（@PreDestroy → DisposableBean.destroy() → 自定义 destroy-method）。
